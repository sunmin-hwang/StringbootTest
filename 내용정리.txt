3. 스프링의 계층구조 : 프레젠테이션 계층 (Controller), 비즈니스 계층 (Service), 데이터 접근 계층 (Repository)
    controller - front한테 요청과 응답 전달(홀서빙)
    service - 비즈니스 로직 처리(주방장)
    repository - db와 소통(식자재 관리자)
    entity - 각 계층들이 주고받는 객체(식자재)

9. Mybatis VS Jpa :
    한국의 IT 환경, 특히 공공기관이나 기존 시스템에서는 MyBatis를 여전히 많이 사용.
    하지만 최신 스타트업과 대기업의 신규 프로젝트는 대부분 JPA를 표준으로.
    Mybatis
    핵심: 개발자가 SQL 쿼리문을 직접 작성하여 자바 객체와 연결.
    장점: 복잡한 통계 쿼리나 데이터베이스에 종속적인 강력한 기능을 세밀하게 튜닝할 때 유리.
    단점: 단순한 CRUD 쿼리도 개발자가 일일이 다 작성해야 함.
         데이터베이스 테이블 구조가 바뀌면 관련된 모든 SQL 문과 자바 코드를 다 수정해야 하는 엄청난 노동이 수반됨.
    Jpa
    핵심: 개발자는 자바 객체(Entity)만 다루고, 복잡한 SQL은 JPA가 알아서 생성하여 실행.
    장점: 객체 지향적인 코드 작성에만 집중할 수 있음.
         단순 반복적인 SQL 작성이 사라져 개발 생산성이 압도적으로 높아지며, 테이블 구조가 변경되어도 자바 코드만 수정하면 끝.
    단점: 초기 학습 곡선이 가파릅니다. 원리를 모르고 사용하면 성능 문제가 발생할 수 있습니다.
